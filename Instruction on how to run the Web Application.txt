Description of the Web Application: Web app developed in Node.js, MongoDB Atlas, Express, Mongoose

The homepage displays all blog posts stored in the database at the bottom. A search bar at the top-right allows querying for specific blog tags or text. On the "Read All Blogs" page, all blog posts are listed, and new posts can be added using the "Add New" button on this page or the dedicated "Add New Blogs" page. Editing or deleting blogs is possible from both the edit/delete page and the "Read All Blogs" page.

Instruction to run the web application

1) Unzip the Assignment_4 file

2) Go inside the Assignment_4 folder and  upload the " blog " folder in VS Code IDE.

3) Install all the dependencies that are in the package.json file through the VS Code terminal. All the installation command given below:

npm i bcrypt connect-mongo cookie-parser dotenv ejs express express-ejs-layouts express-session jsonwebtoken method-override mongoose

List of dependencies with brief descriptions.

bcrypt: For hashing passwords securely.
connect-mongo: MongoDB session store for Express.js.
cookie-parser: Middleware for parsing cookies in Express.js.
dotenv: Loads environment variables from a .env file.
ejs: Embedded JavaScript templating engine for rendering views.
express: Web application framework for Node.js.
express-ejs-layouts: Layout support for ejs in Express.js.
express-session: Session middleware for Express.js.
jsonwebtoken: JSON Web Token (JWT) implementation for Node.js.
method-override: Middleware for HTTP method override in Express.js.
mongoose: MongoDB object modeling tool for Node.js.
nodemon: Development dependency that automatically restarts the Node.js application.

4) In VS Code terminal/Windows Powershell redirect commad line to the specific assignment folder using "C:\Users\........" 

5)type "npm i nodemon --save-dev" and press enter

6)Then type "npm run dev" and the app will run in http://localhost:27017/
*You can the change the local host value in the app.js file based on your preference*
